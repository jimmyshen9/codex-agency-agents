# 审查工作流模板

适用于：PR 审查、补丁审查、发布前风险检查。

把下面这段作为提示词交给 Codex：

```text
使用 engineering-codex-code-review 审查这次改动。

输出要求：
- 先列 findings，再给简短总结。
- findings 按严重程度排序。
- 每条 finding 必须包含：问题、影响、证据位置、建议修复。
- 优先关注：
  - 行为回归
  - 边界条件
  - 并发和状态问题
  - 数据兼容性
  - 缺失测试
- 如果没有发现问题，要明确写“未发现明确缺陷”，并指出剩余风险。
- 全部用中文。
- 不使用 emoji。
```
