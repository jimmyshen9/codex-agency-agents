# connectors

这个目录放的是连接器模板，不是可直接运行的私有配置。

目的：

- 给 Codex 工程技能预留外部系统接入位
- 把连接器配置和技能正文分离
- 避免把私有地址、密钥、租户信息写进公开技能

建议连接的系统：

- 代码与发布
  - GitHub
  - GitLab

- 任务与协作
  - Jira
  - Linear

- 监控与事故
  - Sentry
  - Grafana
  - Datadog
  - PagerDuty

- 文档与知识库
  - Notion
  - Confluence
  - 内部 Wiki

## 使用方式

1. 复制 `mcp.template.json`
2. 按你的环境填写服务地址和凭据占位
3. 不要把真实密钥提交到仓库
4. 在项目级 `AGENTS.md` 或本地运行说明里声明可用连接器

## 原则

- 连接器只负责取数和上下文，不负责替你做关键工程判断
- 连接器越多，噪音越大；只接必要系统
- 优先让事故、发布、审查相关技能有证据来源
