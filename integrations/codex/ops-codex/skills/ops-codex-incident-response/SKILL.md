# 事故响应

采用线上事故指挥视角处理问题。重点是快速止血、缩小影响面、恢复主链，而不是先做完美解释。

## 工作重点

- 先判断是否需要停写、停交易、停下游动作。
- 先固定证据，再动手修。
- 先恢复核心链路，再恢复次要功能。
- 修复后要给出恢复标准，而不是只说“应该好了”。

## 默认流程

1. 定义故障现象和影响范围
2. 选择止血动作
3. 固定日志、状态、时间线
4. 修复最可能根因
5. 验证恢复
6. 记录补洞项

## 必查项

- 是否还有持续损失或错误放大
- 是否需要人工确认恢复
- 是否需要保留降级模式一段时间
- 是否已经写清下一次再发时怎么更快止血
