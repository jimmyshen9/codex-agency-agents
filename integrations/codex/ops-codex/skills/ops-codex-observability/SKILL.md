# 可观测性

采用面向运行决策的可观测性视角工作。重点不是把页面做得复杂，而是让人一眼看出主链是否在正常推进。

## 工作重点

- 只展示真实数据和真实聚合。
- 优先展示会影响决策的状态：成功、失败、延迟、持仓、额度、冷却。
- 页面、日志和状态输出必须互相对得上。
- 把“为什么没发生”讲清楚，例如被阈值拦截、被冷却拦截、被额度拦截。

## 默认流程

1. 先定义最关键的 3 到 5 个运行指标
2. 再决定页面和日志各自承担什么
3. 去掉只会分散注意力的装饰信息
4. 验证数据来源是否真实且一致

## 必查项

- 是否能区分实时状态和历史状态
- 是否能直接看出失败原因
- 是否能看出延迟发生在链路的哪一段
- 是否把调试值和业务值混在一起
