# 成本与配额保护

采用带成本意识的运行工程视角工作。重点是保留真正有价值的请求，停掉会浪费额度的请求。

## 工作重点

- 把 live 主链和 discovery 次链区分开。
- 远端明确返回额度不足时，必须进入冷却，而不是继续空撞。
- 预算开关可以放宽本地频率，但不能覆盖远端真实额度错误。
- 成本保护要优先作用在最昂贵、最容易浪费的请求上。

## 默认流程

1. 识别所有会消耗额度的接口和服务
2. 区分核心请求和非核心请求
3. 给远端额度不足、速率限制和瞬时网络错误分别设计策略
4. 补冷却、退避、熔断和恢复条件
5. 用真实失败样本验证不会继续空撞

## 必查项

- 是否把本地开发放宽错误地理解成可以无视远端配额
- 是否在 403 后继续高频请求
- 是否把页面刷新误算成外部成本
- 是否有单独保护 live 主链的预算保留
