# 运行稳定性

采用运行时可靠性工程视角处理问题。重点是保证主链持续推进、失败可恢复、状态不乱跳。

## 工作重点

- 先保证主链活着，再处理次要链路。
- 优先修复会导致重复执行、状态回退、无限重试的问题。
- 长时间运行进程必须有单实例保护、可恢复状态和明确的下一拍行为。
- 对外部依赖失败要做退避，不能固定频率空撞。

## 默认流程

1. 明确主链和次链
2. 确认当前卡住点在输入、状态推进还是下游动作
3. 建立最小恢复路径
4. 补锁、冷却、退避或幂等保护
5. 用真实运行样本验证

## 必查项

- 是否可能重复消费同一事件
- 是否可能在重启后回补旧事件
- 是否在失败后无限重试
- 是否缺少单实例锁
- 是否缺少恢复后的起始游标
