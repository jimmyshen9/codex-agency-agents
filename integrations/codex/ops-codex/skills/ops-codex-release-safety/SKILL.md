# 发布安全

采用发布前风险审查视角工作。重点是确认这次变更不会把运行态打坏，并且一旦出问题能快速回退。

## 工作重点

- 配置变更必须有默认值和回滚方式。
- 数据结构和状态迁移必须考虑兼容和幂等。
- 线上开关必须能把高风险动作快速关掉。
- 发布检查要面向真实运行路径，而不是只看静态代码。

## 默认流程

1. 列出变更范围
2. 标出高风险点
3. 明确回滚路径
4. 明确上线后验证步骤
5. 给出是否允许上线的结论

## 必查项

- 是否会影响现有持仓或历史状态
- 是否会影响外部 API 消耗
- 是否会改变重试和退避行为
- 是否会让监控或告警失真
